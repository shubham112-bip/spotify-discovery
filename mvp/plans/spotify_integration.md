# Spotify Web API Integration Plan

This document details the OAuth 2.0 flow, scopes, and specific endpoints required to build the Commute Compass MVP.

---

## 1. Authentication Flow

We will use the **OAuth 2.0 Authorization Code Flow** with standard PKCE (or client-secret exchange via backend) to safely obtain a user access token.

* **Redirect URI**: `http://localhost:3001/api/auth/callback` (Local)
* **Production Redirect URI**: `https://<your-app-domain>.vercel.app/api/auth/callback` (Staging/Production)

### Required Scopes

To fetch recommendations and write custom playlists, we need:
1. `user-top-read`: Fetch user's favorite tracks and artists to seed the AI agent.
2. `playlist-modify-public`: Create and write tracks to a public playlist.
3. `playlist-modify-private`: Backup scope to write to private playlists if preferred.

---

## 2. API Endpoints Reference

### Step A: Fetch Taste Profile

Before calling Gemini, the server will retrieve user preferences:

1. **User Profile**:
   * **URL**: `GET https://api.spotify.com/v1/me`
   * **Header**: `Authorization: Bearer <access_token>`
   * **Purpose**: Get User ID (required to create playlist).

2. **Top Artists**:
   * **URL**: `GET https://api.spotify.com/v1/me/top/artists?limit=15`
   * **Purpose**: Identify core favorite genres and artists to guide LLM recommendations.

3. **Top Tracks**:
   * **URL**: `GET https://api.spotify.com/v1/me/top/tracks?limit=15`
   * **Purpose**: Analyze active musical attributes (acousticness, danceability, tempo tags) indirectly through track titles/artists.

---

### Step B: Match AI Recommendations

Gemini returns textual recommendations (e.g. *"Artist: Tame Impala, Song: Borderline"*). The backend will search Spotify to get active track URIs:

1. **Search Catalog**:
   * **URL**: `GET https://api.spotify.com/v1/search?q=track:<song>+artist:<artist>&type=track&limit=1`
   * **Purpose**: Extract the Spotify track URI (`spotify:track:XXXXXX`).
   * **Fallback**: If search returns no results, strip special characters or query artist name only.

---

### Step C: Create Commute Playlist

1. **Create Playlist**:
   * **URL**: `POST https://api.spotify.com/v1/users/{user_id}/playlists`
   * **Payload**:
     ```json
     {
       "name": "Commute Compass: [Mood] Vibe",
       "description": "AI-native playlist generated by Commute Compass based on your current vibe.",
       "public": true
     }
     ```
   * **Response**: Get `id` of the new playlist.

2. **Add Tracks to Playlist**:
   * **URL**: `POST https://api.spotify.com/v1/playlists/{playlist_id}/tracks`
   * **Payload**:
     ```json
     {
       "uris": [
         "spotify:track:XXXXX1",
         "spotify:track:XXXXX2",
         ...
       ]
     }
     ```
   * **Purpose**: Populate the playlist.
